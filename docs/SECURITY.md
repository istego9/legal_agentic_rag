# SECURITY.md

## Принципы
1) Least privilege: агент видит только то, что должен видеть.
2) Pass-through доступ: агент не расширяет права пользователя.
3) Прозрачность: ответы должны сопровождаться предположениями и шагами (и ссылками на артефакты, где возможно).

## Непереговорные правила
- Никаких секретов в репозитории.
- Любые ключи/токены — через секреты CI/секрет-хранилище.
- Команды, которые могут:
  - трогать прод
  - менять инфраструктуру
  - делать network egress
  должны требовать approval (см. `.agentfirst/policies.yaml`).

## Canonical Chunk Model v1 (2026-03-05)
- В search projection (`chunk_search_documents`) не хранить секреты/ключи/сырые токены.
- В metadata проекций хранить только retrieval-полезные поля; не дублировать полный canonical payload.
- Любые поля с потенциально чувствительными данными должны быть nullable и явно whitelisted перед индексацией.
