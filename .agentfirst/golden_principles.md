# Golden Principles (механические правила против энтропии)

Эти правила должны быть **проверяемыми** (линтер/тест), иначе это просто пожелания.

## Примеры (адаптируйте под проект)
1) Shared utilities > локальные хелперы (централизация инвариантов).
2) Валидировать формы данных на границах (ingress/egress).
3) Никаких «скрытых» зависимостей: dependency graph должен быть предсказуемым.
4) Любой публичный API → контрактные тесты.
5) Любое изменение поведения → тест/евал, который падал до фикса и проходит после.

## Garbage Collection
Настройте регулярный процесс (ночные задачи), который:
- сканирует отклонения от принципов
- обновляет `docs/QUALITY_SCORE.md`
- открывает маленькие refactor PR
